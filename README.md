# The Way of Olem Diga

A personal operating manual for a specific mind in a specific world.

**INTP cognitive architecture. Enneagram 5w4. Level 1 ASD sensory and executive profile.**

---

## What This Is

This is not a blog. Not a guide for the general public. It is a personal operating system — a set of protocols, thresholds, and practices designed for one person to navigate a world not built for his wiring.

The site documents a four-stage sequential framework:

| Stage | Focus | Core Question |
| :-- | :-- | :-- |
| I · Foundation | Sovereignty & Self-Knowledge | Can I sustain myself? |
| II · Depth | Shadow Integration & Resilience | What am I capable of? |
| III · Construction | Building & Creating | Am I building something that matters? |
| IV · Legacy | Contribution & Exit | What remains when I'm gone? |

Each stage contains modules with specific practices. Between stages are threshold checks — falsifiable gates that must be passed before proceeding. This is not a buffet. The stages are sequential.

---

## Why This Exists

Standard advice assumes a different operating system. It assumes real-time processing, unlimited social energy, and functional executive ignition. That advice failed repeatedly.

This manual is the result of documenting what actually worked — not what should have worked, not what sounded good. It is a living document, updated as new data arrives.

## Technical

This is a static HTML site with a single CSS file and a single JavaScript file. No build process, no framework.

```

/the-way-of-olem-diga/
├── index.html
├── begin.html
├── anchors/            (philosophical foundations)
├── 01-foundation/      (stage modules)
├── 02-depth/
├── 03-construction/
├── 04-legacy/
├── threshold/          (stage gates)
├── css/way.css
├── js/script.js        (sidebar injection, toggle, progress bar)
└── README.md

```

The design is intentionally minimal. Dark background, single cyan accent, no distractions.

## Usage

This is a personal document. It is written in the first person ("I", "my") because the intended audience is the author. If you find something useful here, you are welcome to take it. But this was not built for you. It was built to keep one specific person walking.
